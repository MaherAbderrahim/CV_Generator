Je vais te donner le contexte complet du projet avant de te demander les corrections, afin que tu comprennes bien l’architecture, l’objectif produit et le fonctionnement global de l’application.

Le projet est une application web moderne de génération de CV développée avec Next.js/React (et probablement Tailwind CSS).
Ce n’est plus un clone basique : l’objectif est maintenant d’en faire une vraie plateforme professionnelle de création de CV premium.

Contexte du produit

L’interface est divisée en deux grandes parties :

Partie gauche : éditeur

À gauche, l’utilisateur remplit les informations de son CV via un système de sections dynamiques.

Exemples de sections :

Informations personnelles
Contact
Expérience professionnelle
Formation
Compétences
Langues
Projets
Certifications
Réseaux sociaux
etc.

L’utilisateur peut :

ajouter des sections,
supprimer des sections,
réorganiser les sections,
modifier les contenus en temps réel.

Il existe aussi plusieurs options de personnalisation :

changement de couleurs,
changement de polices,
taille de police,
espacement/interligne,
thèmes,
designs/templates différents,
etc.
Partie droite : preview temps réel

À droite, on a un aperçu live du CV.

Toutes les modifications faites dans l’éditeur doivent être reflétées instantanément dans le preview :

texte,
couleurs,
typographie,
espacements,
templates,
sections,
etc.

Le projet possède déjà plusieurs templates/designs de CV, avec un système de personnalisation dynamique.

Le CV doit également pouvoir être exporté proprement en PDF avec un rendu fidèle au preview.

Maintenant que tu as le contexte complet du projet, voici les problèmes et améliorations que je veux que tu traites.

# 1. Bug : changement de taille de police qui ne fonctionne pas

Actuellement, quand je change la taille de la police (petit / moyen / grand), rien ne se passe dans le CV.

Tu dois :

* Identifier pourquoi le paramètre ne se propage pas.
* Vérifier :

  * le state,
  * les props,
  * le context/store,
  * les variables CSS,
  * les classes Tailwind,
  * les styles inline,
  * et les composants templates.
* Vérifier si la valeur change réellement dans le state mais n’est pas appliquée visuellement.
* Corriger le binding complet.
* Faire en sorte que le changement soit :

  * instantané,
  * visible en temps réel,
  * cohérent sur tous les templates,
  * sauvegardé,
  * et compatible avec l’export PDF.

Je veux un vrai système scalable de typography settings, pas juste un patch rapide.
